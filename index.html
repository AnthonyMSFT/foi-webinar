What we’re building

A semi-autonomous declarative agent, built in Microsoft Copilot Agent Builder. No code, no infrastructure, no separate platform to manage — the agent lives inside the Microsoft 365 tenant you already have, governed by the controls you already use.

No-code

Declarative, not bespoke

Configured through Agent Builder: instructions, knowledge sources, a handful of guard-rails. If your guidance changes, you update the agent — not a development backlog.

Semi-autonomous

Drafts — humans decide

The Process

We break a typical FOI workflow into three stages. Copilot is well suited to stages 1 and 3 — the parts that involve reading guidance and drafting language. Stage 2 stays with your people, because it needs access to data that the requester (and Copilot acting on their behalf) doesn’t have. The timings below come from research with councils and police forces and reflect what a seasoned Public Access colleague typically spends on each stage.

Stage 1

Triage request

Check the request against ICO and sector guidance. Identify exemptions, PII, vexatious or overly broad asks.

~20 mins per request
  • Decide if it’s an FOI, SAR or something else
  • Route to the correct team
  • Apply any exemptions
  • Check if it’s already been answered
45% of police FOIs can be exempted at triage

Stage 2

Action request

Gather information across the organisation. Requires privileged access far beyond a single user’s permissions.

~3 hours weighted average
  • Researching and finding all the data
  • Highly variable — some asks are easy, some are hard
  • Average weighted across easy / medium / hard requests

Stage 3

Respond to citizen

Draft a clear, compliant reply — whether accepting, refusing, or asking for clarification.

~1 hour per response
  • Collate all the data gathered in stage 2
  • Write the response
  • Any redaction required
  • Approval where needed
  • Paperwork — recording the response

Why Copilot can’t do stage 2 (and why that’s OK)

Copilot operates with the permissions of the signed-in user. An FOI response usually needs a privileged, organisation-wide view of a subject. That’s a job for your established FOI process and the people authorised to run it. The agent’s value is removing the repetitive reading, classifying and drafting work that surrounds it.

Reads the same guidance you do

Point it at the ICO, College of Policing, or your own SharePoint policy library. It applies the same rules, consistently.

Lives where the work arrives

Triage happens inside Outlook on the incoming email — no copy-pasting, no new tools to learn.

Hands off cleanly

Qualified requests are passed to your existing FOI process untouched. Nothing is auto-sent to citizens.

Drafts, never decides

For rejections and exemptions, the agent prepares a draft reply for a human to review, edit and approve.
