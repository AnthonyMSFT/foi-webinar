Build the FOI agent

Follow these steps in Microsoft 365 Copilot Agent Builder. Everything is declarative — no code, no Azure, no admin approvals beyond your normal Copilot agent publishing process.

Start at step 1 Open the prompt library

Before you start

You will need:
  • A Microsoft 365 Copilot licence with Agent Builder enabled.
  • Permission to publish agents to yourself (we won’t deploy organisation-wide today).
  • One or two real (or realistic) sample FOI requests to test against.
  • The public URL of the guidance you want the agent to follow — e.g. the ICO FOI hub or your sector regulator.

The build, step by step

Each step takes roughly 2–5 minutes. Copy prompts directly from the dark blocks.

1

Open Copilot Agent Builder

In Microsoft 365 Copilot, choose Create agent. Pick the Custom option so you can write your own instructions and add knowledge sources.

  • Name: FOI Triage Assistant
  • Icon: choose anything recognisable — we’ll come back to branding later.
2

Write the agent description

The description is what colleagues see when they pick the agent. Keep it crisp and scope-bound.

Helps Public Access and FOI colleagues triage incoming Freedom of Information requests against ICO guidance and our internal policies. Identifies likely exemptions, PII risks, vexatious or overly broad requests, and drafts citizen responses for refusals. Does not action requests or send replies automatically.
3

Add the system instructions

This is the heart of the agent. It defines the triage → hand-off → draft pattern and the guardrails.

Purpose

Manage incoming FOI and SAR requests to the UK Department for Copilot and AI.

Response format

Always reply using the following strict format and do not add any extra information:

  • Classification: FOI / SAR / Other
  • Action: Action / Exempt
  • Justification: If exempt, provide a short (up to 100 words) justification. When exempting, always cite relevant legislation and highlight it in bold within the justification.

Guidelines

  • Review the content of each request and classify as FOI, SAR, or Other.
  • Consult ICO guidance from provided URLs for each classification.
  • Decide whether to Action or Exempt the request based on ICO standards.
  • Provide clear justification for any exemptions, always citing and highlighting the relevant legislation in bold.

Example

Given an email: “Please send me all data you hold about me.”

  • Classification: SAR
  • Action: Action
  • Justification: N/A

If a request is unclear, ask for clarification in the same strict format.

4

Add knowledge sources

In Agent Builder, add Knowledge → Public website for each authoritative source. Start narrow — quality beats volume.

  • https://ico.org.uk/for-organisations/foi/ — ICO FOI hub
  • Your sector regulator (e.g. College of Policing APP on Information Management for forces; relevant guidance for HMRC, councils, NHS trusts, etc.)
  • Optional: a SharePoint site or document library containing your internal FOI handling policy
Tip: Tell colleagues that only public sources are indexed today. Adding SharePoint content is fine, but be deliberate — the agent will inherit each user’s permissions on those files.
5

Add starter prompts

Starter prompts appear in the agent’s welcome screen. They cue the right behaviour and make the agent feel purpose-built.

LabelPrompt
Triage this requestTriage the FOI request in this email. List any likely exemptions, PII concerns and broadness issues, with citations.
Is this a SAR?Tell me whether the request in this email is a Subject Access Request rather than an FOI request, and explain why.
Draft a refusalDraft a polite citizen-facing refusal for this request, citing the relevant FOIA section and the ICO guidance.
Ask for clarificationDraft a short reply asking the requester to narrow the scope so we can respond within the cost limit.
6

Test against sample requests

Open the agent in Outlook on a sample email and run each starter prompt. Look for three things:

  • Citations — every exemption claim links back to ICO or sector guidance.
  • Hand-off discipline — for accepted requests, it stops and refers to your FOI process.
  • Tone — drafts are polite, plain English, UK spelling, no jargon.

Use the sample requests on the resources page if you don’t have one handy.

7

Iterate the instructions

Common tweaks after first testing:

  • Force a structured output (e.g. Recommendation, Reasoning, Citations, Suggested next step).
  • Add an explicit list of exemptions your team sees most often.
  • Tell it to refuse to draft anything until the officer confirms the triage outcome.
8

Publish to yourself, then to your team

Publish privately first. Once you’re happy, ask your Copilot admin to approve the agent for your FOI team. Keep the scope small — this is a power tool for trained officers, not a public-facing chatbot.

Governance reminder: Log the agent in your AI register, capture an owner, and schedule a review date. The ICO expects organisations to know which AI tools are touching information-rights work.
